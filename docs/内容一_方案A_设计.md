# 内容一 · 方案A：结构分支增强的复杂表格识别（模型改造）

> 在 Qwen2.5-VL 基线上做架构级改造，针对社保无线表 / 复杂表头 / 合并 / 空单元格。

## 1. 动机（baseline 实测支撑）

baseline（Qwen2.5-VL-3B + LoRA → HTML）在 test(320) 上：

| 指标 | 值 |
|---|---|
| TEDS / S-TEDS | 0.954 / 0.957 |
| 完全匹配率 | **0.078** |
| 行数准确率 | **0.528** |
| 列数准确率 | 0.844 |
| 合并单元格 span F1 | **0.487** |
| 空单元格 F1 | **0.553** |
| 单元格内容准确率 | 0.528 |

结论：**TEDS 平均分高，但结构细节（行数、合并跨度、空单元格）系统性出错**，整表几乎不能 100% 还原。这正是需要改进的点。

## 2. 总体思想

在 Qwen2.5-VL 的多模态解码器之上**并联一个轻量"结构分支"**，显式预测表格逻辑结构（占用网格、合并跨度、空标志、表头层级），并与主"内容(HTML)解码"做**双向互学习**互相约束。推理仍是单个端到端模型。

```
表格图 → 视觉编码器 → 多模态解码器 ──┬─→ 主分支：HTML(结构+内容) 序列
                                   │        ↑↓ 双向互学习/一致性
                                   └─→ 结构分支：占用网格 / 合并跨度 / 空标志 / 表头层级图
```

## 3. 模块设计

- **M1 结构解码分支**：取主解码器若干层的 hidden states → 抽取单元格表示 → 预测：(a) 行/列归属与占用矩阵，(b) rowspan/colspan，(c) 空/非空，(d) 单元格角色。借鉴 TDATR（用语言解码器隐状态做结构）、UniTabNet（逻辑解码器）。
- **M2 表头层级图**：对表头单元格预测**父子边**与**每个数据列的表头路径**（轻量图注意力/分类头）。**这是 UniTabNet/TDATR 等都没做的差异点。**
- **M3 结构↔内容双向互学习**：结构分支与内容解码共享编码、互为辅助监督，并在两者结构不一致处加一致性损失。借鉴 Multi-Cell Decoder 的 mutual learning。
- **C 约束解码（组件，非主创新）**：用 M1 的占用状态在生成时屏蔽非法跨度/越界 token，把"约束"从事后修补变成内建。

## 4. 训练目标

```
L = L_html(主HTML) + λ1·L_occ(占用) + λ2·L_span(合并跨度)
        + λ3·L_empty(空单元格) + λ4·L_header(表头父子边/路径) + λ5·L_consist(结构↔内容一致)
```

## 5. 与现有工作的边界

| 论文 | 它做了什么 | 借鉴 | 我的不同 |
|---|---|---|---|
| UniTabNet (EMNLP'24) | 物理/逻辑双解码器 | 结构-内容双分支 | 做在预训练文档 VLM 上；加表头层级图 |
| TDATR (2026) | 结构引导模块用解码器隐状态 | 隐状态→结构表示 | 不回归 bbox；改为占用/跨度/空+表头层级 |
| Multi-Cell Decoder (2024) | 结构/内容双解码 + 双向互学习 | 双向互学习机制 | 面向无线/合并/空 + 表头层级一致性 |
| Hierarchical Modeling (2025) | 多任务+非因果 refiner | 多任务监督 | 显式表头层级树 + 结构内容互约束 |

公开文献明确指出：**"在 VLM backbone 隐表示上挂辅助结构解码器做多任务"在文档分析领域尚未普及、机制尚未被充分解释**——这是一个有先例、可实现、又有新意空间的开放方向。

## 6. 实验设计

- **主对比**：方案A vs baseline，指标 = span F1 / empty F1 / **header-path 准确率（baseline 无法产出）** / 行列准确率 / TEDS / 完全匹配率。
- **消融**：逐个去掉 M1 / M2 / M3 / 约束解码。
- **分组**：按 有线/无线、表头层数、宽表 分别报（需补无线表样本进 test）。

## 7. 工程落地与风险

- 实现路径：很可能不走 ms-swift 现成 pipeline，而是用 transformers + peft 自定义训练脚本，包装 Qwen2.5-VL 并挂结构头与多任务损失（底座 LoRA、新模块全量训）。
- 风险：给 VLM 加分支要改 forward，工作量中等、调试期较长。
- 分步降险：**先加 M1 + 跨度/空损失 → 看 span F1 / empty F1 是否提升 → 再加 M2(表头层级) / M3(互学习) / 约束解码**，每步都与 baseline 对比。
